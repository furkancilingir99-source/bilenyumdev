BILENYUM — SUPABASE GİRİŞ SİSTEMİ KURULUMU
==========================================

Kullanıcıları Supabase panelinden sen yönetirsin. Şifreler hash'lenir; repoda/sitede görünmez.


ADIM 1 — Supabase (tamamlandıysa atla)
---------------------------------------
1. https://supabase.com → proje: bilenyum
2. Authentication → Users → Add user → Create new user
3. ✓ Auto Confirm User (işaretle — zorunlu)
4. E-posta + şifre belirle → Create user


ADIM 2 — API anahtarlarını kopyala
-----------------------------------
Supabase Dashboard → Project Settings → API

  SUPABASE_URL        = Project URL  (https://xxxxx.supabase.co)
  SUPABASE_ANON_KEY   = anon public key  (eyJhbG... ile başlar)


ADIM 3 — Vercel'e ekle (ZORUNLU — site bunu bekliyor)
------------------------------------------------------
https://vercel.com → bilenyumdev → Settings → Environment Variables

  Name: SUPABASE_URL        Value: (Project URL)
  Name: SUPABASE_ANON_KEY   Value: (anon public key)

  Environment: Production ✓
  Save

Sonra: Deployments → son deploy → ⋯ → Redeploy


HIZLI YOL (terminal — Vercel girişi gerekir)
---------------------------------------------
PowerShell:

  cd "C:\Users\Furkan\Downloads\webnew-main (4)\webnew-main"
  .\scripts\set-vercel-env.ps1 -SupabaseUrl "https://xxxxx.supabase.co" -AnonKey "eyJhbG..."


ADIM 4 — Giriş testi
---------------------
https://devbilenyum.vercel.app/giris

Supabase'de oluşturduğun e-posta + şifre ile giriş yap.


KULLANICI YÖNETİMİ (Supabase Dashboard)
----------------------------------------
https://supabase.com/dashboard → Authentication → Users

  Yeni ekle:     Add user → Create new user → ✓ Auto Confirm User
  İptal / sil:   Delete user
  Şifre sıfırla: Send password recovery
  Oturum kapat:  Sign out user


ÇIKIŞ
-----
Profil menüsü → Çıkış Yap
veya: https://devbilenyum.vercel.app/api/logout


OTURUM SÜRESİ
-------------
• Giriş yaptıktan sonra tarayıcı oturum işareti (localStorage) oluşturulur.
• Tarayıcı tamamen kapatılıp yeniden açılınca işaret yoksa eski çerez silinir → tekrar giriş gerekir.
• Son açık sekme kapanırken de çıkış yapılır.
• F5 / Ctrl+R yenilemede oturum korunur.
• Supabase JWT süresi dolunca (≈1 saat) yine giriş gerekir.

Not: Eski oturumdan kalan çerez varsa ilk ziyarette otomatik temizlenir; bir kez tekrar giriş yapmanız yeterli.

Supabase JWT süresini 3600 saniye (1 saat) yapın:
  Authentication → Settings → JWT expiry → 3600


SORUN GİDERME
-------------
• Site 503 gösteriyor → Vercel env eksik, ADIM 3
• "E-posta onaylanmamış" → Supabase'de Auto Confirm ile yeniden oluştur
• "Şifre hatalı" → Supabase'deki e-posta/şifre ile aynı mı kontrol et

Supabase panel: https://supabase.com/dashboard
